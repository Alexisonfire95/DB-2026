# Звіт до Лабораторної роботи №4
## Тема: Аналітичні SQL-запити (OLAP)

### Реалізований функціонал
Було розроблено та протестовано набір аналітичних запитів для отримання бізнес-статистики(працювали удвох тому кількість запитів удвічі більша за рекомендовану):

* **Агрегація та Групування (8 запитів):**
    * Використано функції `COUNT`, `SUM`, `AVG`, `MIN`, `MAX` для отримання базових метрик (кількість користувачів, активні товари, ціновий діапазон).
    * Застосовано `GROUP BY` для аналізу наповненості категорій, статусів замовлень та складських запасів.
    * Реалізовано фільтрацію згрупованих даних через `HAVING` (пошук популярних категорій).
    * Виконано роботу з датами (`::DATE`) для аналізу динаміки реєстрацій.

* **Об'єднання таблиць - JOIN (6 запитів):**
    * Використано `INNER JOIN` для створення людино-читабельних звітів (назви категорій замість ID, деталі замовлень).
    * Використано `LEFT JOIN` для пошуку сутностей без пари (наприклад, товари без продажів або користувачі без магазинів).
    * Реалізовано мульти-джойн (об'єднання 4 таблиць) для формування повної картки замовлення.

* **Вкладені запити - Subqueries (6 запитів):**
    * **Скалярні підзапити:** Фільтрація замовлень вище середнього чека.
    * **IN / NOT IN:** Пошук покупців певної категорії та товарів, які ніколи не купували ("товари-привиди").
    * **Корельовані підзапити:** Пошук найдорожчого товару всередині кожної окремої категорії та останньої дати замовлення для клієнта.
    * **Derived Tables (підзапит у FROM):** Розрахунок середньої кількості одиниць товару в чеку через проміжну віртуальну таблицю.
