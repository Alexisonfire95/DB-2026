# Звіт до Лабораторної роботи №3
## Тема: Маніпулювання даними SQL (DML) та Транзакції (OLTP)

### 1. Структура бази даних
Для виконання завдань цієї лабораторної роботи ми продовжили використовувати схему бази даних Маркетплейсу створеної в рамках попередньої **Лабораторної роботи №2**

**Структура таблиць, типи даних та зв'язки (Primary/Foreign Keys) залишилися незмінними.**
Жодних змін до DDL (структури) не вносилося, оскільки мета роботи полягала у перевірці цілісності даних та реалізації бізнес-логіки через DML-операції.

### 2. Реалізований функціонал
В ході роботи було написано SQL-скрипти для імітації реальних процесів магазину:
* **CRUD операції:** Реєстрація користувачів, додавання товарів, оновлення цін (із використанням умов `WHERE`) та видалення відгуків.
* **Транзакція:** Реалізовано атомарну транзакцію оформлення замовлення, яка включає:
    1.  Створення замовлення.
    2.  Додавання позицій із фіксацією поточної ціни.
    3.  Автоматичне списання залишків товару зі складу (`stock_quantity`).
    4.  Автоматичний розрахунок суми замовлення.
    5.  Копіювання адреси доставки з профілю користувача.
